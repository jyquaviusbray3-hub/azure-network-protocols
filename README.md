In this step, I navigated to the official Wireshark website to download the tool that will be used for network traffic analysis. From the download page, I selected the Windows x64 Installer for the latest stable release, which in this case was version 4.4.3. Since the virtual machine I’m using runs a 64-bit version of Windows, this installer is the correct option. Wireshark will be used later to capture and analyze network packets, which is useful for monitoring traffic and troubleshooting network-related issues. After selecting the installer, the file begins downloading so it can be installed on the system in the next step.

In this step, I used the ping command in PowerShell to test network connectivity between machines on the network. I ran the command ping 10.0.0.4 from the system, which sent ICMP echo requests to that IP address. The replies confirmed that the connection was successful, with no packet loss and response times of around 1–2 milliseconds. At the same time, I had Wireshark running to capture and analyze the network traffic generated by the ping command. In Wireshark, I filtered for ICMP traffic, which allowed me to clearly see the echo request and echo reply packets being exchanged between the two systems. This helped demonstrate how network traffic can be captured and inspected in real time while performing basic connectivity tests.
